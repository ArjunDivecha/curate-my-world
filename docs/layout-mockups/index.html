Top Screen Layout Mockups

Open each option, then resize the browser narrow to see the mobile layout. These are static mocks (no app code changes).

Option A
Command Bar + Chips

One compact “command” row: search, date, quick filters. Category chips become horizontally scrollable.

Best for: mobile, fast scanning, fewer vertical blocks.
Option B
Hero + Stats + Category Rail

A small hero with 2–3 stats (events, venues, last refresh), then search and a category rail.

Best for: reassuring “it’s alive” + freshness; still compact.
Option C
Sticky Controls (Mobile)

Top controls become sticky on scroll, so filters/search are always one tap away.

Best for: heavy browsing sessions on iPhone.
Recommendation: Option A for simplicity + smallest footprint. If you want visible freshness, Option B. If you scroll a lot on mobile, Option C.
